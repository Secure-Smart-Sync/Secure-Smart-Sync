## Secure-Smart-Sync (SSS) — 

### What this project is
Obsidian plugin syncing vaults to **Cloudflare R2** with **client-side AES encryption**. Built from a stripped-down fork of "remotely-save". Uses esbuild, TypeScript, AWS S3 SDK via Obsidian's `requestUrl()` to bypass CORS.

### Key architecture decisions (current state)

**Encryption stack:**
- **Only remote is wrapped** with `StorageEncrypt`. Local is always plain `StorageLocal`. The old code wrapped both sides, which caused local plaintext keys to be fed into the decryptor — root cause of the original crash.
- `openssl-base64`: encrypts file **content** and file **names**, but folders are stored **plaintext** on R2 (S3 folders are synthetic key prefixes, no name to encrypt)
- `rclone-base64`: encrypts both file names AND folder names
- `isPlaintextPassthrough(key, method)` helper gates folder names out of encrypt/decrypt for openssl-base64 throughout `storage-encrypt.ts`
- `validatePassword()` now reads from `inner.walkPartial()` directly (bypasses decryption), filters to file-only entries before sampling, so folder-only remotes don't false-fail

**Sync engine:**
- Cache-less first sync: files only on one side → push/pull; both sides no history → conflict resolution
- ignorePaths: glob matching (*, **, directory prefix, exact) applied during `buildMixedEntities`
- Conflict backup: losing side saved as `foo.conflict-YYYY-MM-DD.md` before overwrite (non-fatal if backup fails)
- Retry: 3 attempts, 500ms/1s/2s exponential backoff per task
- prevSync cleanup on delete: `deletePrevSyncRecord` called after successful deletes
- prevSync update: re-stats local after push/pull to capture actual post-write metadata

**DB:** `sss_db` (was `r2sync_db`), namespaced by vaultId
