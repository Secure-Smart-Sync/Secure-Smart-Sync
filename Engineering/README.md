### What? 

The `Engineering` folder contains all the relevent documentations regarding the Technical and problem solving aspect of this obsidian plugin, detailing decisions made throughout the process and reasoning behind those, along with compromises and tradeoffs made in the architecture for the sake of things I consider 'uncompromizable'.

one of the earliest realization during the development of the plugin was, obsidian mobile and obsidian on pc are fundamentally quiet different when it comes to user experience and user interface, this realization early on prompted me to work on seperate plugin for mobile which is fully compatible with desktop plugin...but it turned out, it was a overkill to make entirely different plugin, although it wouldve allowed for more fine tuned experience, the development process itself would not be feasible for a solo dev...even with agentic ai coding. 
hence i decided to take the middle ground...the sss status indicator on mobile is one of the example mobile specific ux.

the sss relay is another one of the core engineering decision made in order to facilitate quick set-up, but trust was crucial for something that dealth with credentials. hence the code of the sss relay is also licensed open source on this account "xensenx". 

there are many such examples of decisions made for engineering and solving core problems. this folder keeps the record of those along with other raw reference materials.
